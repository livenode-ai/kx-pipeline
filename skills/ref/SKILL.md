# /ref — ナレッジ参照スキル

notes/ のナレッジを段階的に参照し、コンテキスト消費を最小化する。

## 手順

### Step 1: INDEX で概要把握

まず `notes/CLAIMS_INDEX.md` を Read する。
このファイルには全ノートの主張（Claim）が1行ずつ集約されている。

質問に関連しそうなノートを CLAIMS_INDEX から選定する（通常3-5件）。

### Step 2: 主張セクションで絞り込み（構造化ノートのみ）

選定したノートに `## 主張（Claim）` セクションがある場合：
- `Grep` で `## 主張` セクション（-A 3）だけ取得する
- この時点で十分回答できるなら、ここで止める（全文Readしない）

### Step 3: 必要なノートだけ深掘り

根拠やデータが必要な場合のみ、該当ノートを Read で全文読む。
複数ノートを比較する場合も、まず主張セクションで候補を絞ってからReadする。

## 出力形式

回答後、参照したノートを末尾に記載する：

```
参照: [[ノートタイトル1]], [[ノートタイトル2]]
```

## 重要

- CLAIMS_INDEX.md が存在しない場合は、通常のGrep/Readで代替する
- 1ファイルだけ参照する場合でも、まずINDEXで文脈を確認してからReadする
- ユーザーの質問に対して「知らない」と答える前に、必ずINDEXを確認する
